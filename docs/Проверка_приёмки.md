## ⏳ Требует проверки на оборудовании музея (нельзя на dev-машине)

Эти пункты проверяются при удалённой установке по видеосвязи с IT музея:

- [ ] **Запуск в киоске на реальном Astra Linux** (Firefox `--kiosk` / Chromium
      `--kiosk --app`), полный экран на проекторе **1920×1080**, читаемость UI.
- [ ] **Автозапуск** (`install-autostart.sh`) после перезагрузки + watchdog-перезапуск.
- [ ] **Реальные геймпады Xbox/XInput по Bluetooth**: стройка — до 4 одновременно,
      викторина — 2 команды; привязка по индексу, все кнопки.
- [ ] Отключение гашения экрана/сна на конкретном ноуте.
- [ ] Стабильность на сессии 1–2 часа (нет утечек/просадок FPS на RTX 3050).

## 📦 Сборка поставки (C2)

Финальные папки по контрактам собираются скриптом:
```bash
./build-delivery.sh
```
Создаёт `delivery/contract1-stroika/` и `delivery/contract2-viktorina/` — в каждой
готовая сборка + исходник (HTML — это и сборка, и исходник) + `libs/` + `kiosk/` +
`docs/`. Перед отправкой заказчику руководства при необходимости конвертируются в PDF.

> Состав поставки по каждому контракту (по ТЗ): готовая сборка + исходный код +
> инструкция по установке + руководство пользователя. Для самодостаточного
> HTML-билда «сборка» и «исходный код» — это один и тот же файл.
