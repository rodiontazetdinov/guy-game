## 2. Платформа и ЖЁСТКИЕ ограничения

- Запуск на **ноутбуке музея под Astra Linux** (сертифицированная российская ОС на Linux), в браузере **Firefox / Chromium** в режиме **киоск (полный экран)**.
- **ПОЛНЫЙ ОФЛАЙН.** На машине музея нет интернета. → **ВСЕ зависимости должны быть локальными/вшитыми**: Three.js, GLTFLoader, шрифты, 3D-модели, звук. Сейчас часть тянется с CDN (cdnjs, jsdelivr, Google Fonts) — **это надо убрать и положить локально**.
- Вывод на **проектор Full HD 1920×1080** — крупный читаемый UI.
- Управление — **геймпады Xbox/XInput по Bluetooth** (у музея 30 шт). Реально одновременно 4–6 (предел Bluetooth-радио), UX-оптимум — 4. Викторина — **2 команды = 2 геймпада**.
- Резервное управление — клавиатура. **Клавиши читать по `e.code`** (физические, не зависят от раскладки!) — уже сделано, НЕ ломать (на русской раскладке `e.key` даёт «цфыв» вместо WASD).
- Железо: i5-12450H, 16 ГБ, RTX 3050 — но игра в браузере (WebGL), поэтому держать **производительность**: тяжёлые фотоскан-модели не использовать.

## 3. Визуальный стиль

- Советский город 1960–70-х: тёплая сепия, кирпичные хрущёвки, копёр шахты, монумент со звёздами и серпом (есть архивное фото-референс города).
- Уровень графики — **«стилизованный реализм»**: текстурированные **game-ready** модели (ассет-сторы, напр. Sketchfab) + low-poly. 
  - НЕ фотореализм (не лезет в бюджет/производительность).
  - НЕ плоский мультяшный low-poly (заказчику **не нравится**).
  - ТЗ требует «низкополигональную графику» → текстурные модели идут **выше** требований, это безопасно для приёмки.
- Шрифт — скрипт в духе советского кино (сейчас Lobster). Для офлайна **вшить локально** (.woff2 в base64 или файлом рядом).
- **Лицензии ассетов:** модели по CC-BY (напр. *Soviet Panel House* © onedev46, Sketchfab) → обязателен экран **«Авторы / Credits»** с указанием авторов. Уже добавлен на стартовом экране стройки — поддерживать при добавлении новых ассетов.

## 4. Текущее состояние (что уже сделано)

**Игра-стройка** — рабочий прототип `gay-stroika-prototype.html`:
- 7 этапов освоения (реальная история Гая): степь → палаточный городок → шахта (копёр)/карьер → ЛЭП/железная дорога/пруд → жилые дома с ростом этажности 2→3→4→5→9 → школы/детсады → парки/Дворец культуры/Дворец спорта → памятники.
- Локальный кооп **2 игрока** (геймпад + клавиатура), панель доступных объектов, баннеры смены этапа, всплывающие исторические справки (**тексты ЧЕРНОВЫЕ, заменить на музейные**), финальный экран статистики.
- Дом «5 этажей» = реальная 3D-модель (`panel-house.glb`), вшита через `panel-house-data.js` (base64 — чтобы работало офлайн и при открытии file://).
- Стек: Three.js r128 + GLTFLoader 0.128 + шрифт Lobster — **сейчас с CDN, локализовать**.

**Вспомогательное:**
- `model-viewer-referens.html` — просмотрщик 3D-модели (референс графики для заказчика).
- `panel-house-data.js` (~10 МБ base64) + `panel-house.glb` (8 МБ) — модель хрущёвки.

**Викторина** — ещё НЕ начата (есть только вопросы и согласованный формат).

## 5. Контент от музея (статус — важно)

- **Игра-стройка:** финальные выверенные тексты исторических справок и архивные фото — даёт музей (присланы не все). Фоновое фото города `gay-fon.jpg` обещано (когда будет — положить рядом, подхватывается на стартовом/финальном экране и горизонте сцены).
- **Викторина:** вопросы есть — 2 набора: «Творчество Ю. Г. Шевцова» (10 вопросов) и «История города Гая» (14+). Формат — 4 варианта (А/Б/В/Г).
  - 🚩 **БЛОКЕР:** в вопросах **не отмечены правильные ответы** и **не присланы фото** к вопросам. Запрошено у музея. Без этого викторину не доделать — структуру и движок строить можно, контент подставить потом.

## 6. Формат викторины (согласован с заказчиком)

- **2 команды = 2 геймпада.** На экране — вопрос + фото + 4 варианта (А/Б/В/Г).
- Команды отвечают кнопками; за правильный ответ — **звёздочки: красные одной команде, жёлтые другой**.
- Ведётся счёт, в конце — таблица результатов и команда-победитель.
- Тот же советский стиль, киоск, офлайн.

## 7. Поставка и приёмка

- **Удалённая установка** (исполнитель НЕ едет в Гай): игра на флешке почтой + настройка по видеосвязи с IT-специалистом музея. Нужен его контакт.
- Состав поставки по каждому контракту: **готовая сборка + исходный код + инструкция по установке (киоск на Astra) + руководство пользователя**.
- Приёмка: демонстрация по видео + тест музея на своём оборудовании; затем подписание накладной (УПД).

## 8. Конвенции разработки

- Каждая игра — **самодостаточный билд**, работает **офлайн**, открывается двойным кликом И через киоск-скрипт.
- **Никаких сетевых зависимостей.** Всё локально.
- Не ломать рабочий ввод: геймпады + клавиатура по `e.code`.
- UI и комментарии — **на русском**.
- Крупный, читаемый с проектора интерфейс; рассчитан на детей 6–14.
